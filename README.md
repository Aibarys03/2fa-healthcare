# Few-Shot 2FA — Face Recognition + OTP

Двухфакторная аутентификация на основе CNN с Triplet Loss.  
Работает с **5–10 фото на пользователя**. Без GPU — только CPU.

---

## Установка

```bash
# 1. Клонируйте / распакуйте проект
cd 2fa_project

# 2. Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate        # Linux/Mac
venv\Scripts\activate           # Windows

# 3. Установите зависимости
pip install -r requirements.txt
```

---

## Быстрый старт

### Шаг 1 — Подготовьте данные

Создайте папки и добавьте фото:

```
data/users/
  user_1/          ← 5-10 фото одного человека
    photo1.jpg
    photo2.jpg
    ...
  user_2/
    photo1.jpg
    ...
```

Минимум **2 пользователя**, минимум **2 фото** у каждого (рекомендуется 5–10).

### Шаг 2 — Запустите веб-интерфейс

```bash
python app.py
```

Откройте браузер: **http://localhost:8000**

### Шаг 3 — Через интерфейс:

1. **Данные** → загрузите фото пользователей
2. **Данные** → запустите обучение (20–30 эпох)
3. **Регистрация** → создайте биометрический профиль + получите QR для OTP
4. **Верификация** → пройдите 2FA (лицо + код из аутентификатора)

---

## Обучение через терминал

```bash
python train.py --data_dir data/users --epochs 30
```

Параметры:
- `--data_dir` — путь к папке с пользователями
- `--epochs` — количество эпох (20–50 для малых данных)
- `--save_dir` — куда сохранять модель (по умолчанию `models/`)

## Технические детали

| Компонент | Описание |
|-----------|----------|
| CNN архитектура | 4 conv-блока + FC, 128-мерный эмбеддинг |
| Функция потерь | Triplet Loss (margin=0.3) |
| Оптимизатор | Adam (lr=0.001, cosine scheduling) |
| Порог верификации | Cosine similarity ≥ 0.70 |
| OTP | TOTP (RFC 6238), окно ±1 интервал |
| Инференс | CPU: ~250–400 мс/изображение |

## Устранение проблем

**Модель не загружается:**  
→ Проверьте что файл `models/best_model.pth` существует. Запустите обучение.

**Низкая точность верификации:**  
→ Добавьте больше разнообразных фото (разное освещение, углы). Увеличьте число эпох.

**OTP не принимается:**  
→ Проверьте время на вашем устройстве — TOTP чувствителен к рассинхронизации.

**Ошибка "минимум 2 пользователя":**  
→ Triplet Loss требует минимум 2 разных пользователей для формирования триплетов.
